Your Coding Environment

Although it isn't necessarily all that well covered in most introductions to programming, the environment you have set up to program in is very important. As there is a sea of possibilities and options out there, choosing something you are comfortable working with is extremely important. It is important that writing code, compiling code, changing code, refactoring, testing and running are all simple tasks, and with a good environment should never be a source of frustration as that in turn often leads to sloppiness.

Coding Environment by Components

Editor

The first thing one needs to start programming is a good editor. As most of your time programming, outside of planning and preparing is in your editor, writing code. A good editor facilitates programming, and makes writing correct programs easy.

An absolute minimum requirement for any editor is syntax highlighting, meaning that the editor colour text based on context. The next step would be autocompletion of functions and interfaces and possibly continuous syntax checking. These make sure that you more reliably write correct programs, and that you do not fail to compile your program simply because of a typo. Finally, when one gets more comfortable with the language one are using and various paradigms of programming, automatic code generation will save you from wasting time on typing out lengthy syntax.

All of the above features should be available in any good IDE (which we will talk about more below), but it is also attainable with a good customizable editor. The two most popular and highly customizable text editors are vim and emacs. As I am a vim user myself, I will quickly describe the vim setup I use when programming C++.

Vim setup

Bare vim is by itself an excellent editor, but without plugins, it does not support all of the properties we want from out editor. Vim comes with syntax highlighting from before, but even that can be improved upon. There are many resources out there to learn to love vim, so I will only focus on the plugins I use on a daily basis.

  • vim-cpp-henhanced-highlight

    An improvement to the C++ syntax highlighting. It contains new keywords introduced by the C++11/14, as well as keywords for STL and boost.

  • YouCompleteMe

    The be-all and end-all vim autocomplete plugin. If set up with semantic support for C-family languages it will do context sensitive autocompletion as well as on the fly syntax checking.

  • ultisnips

    Set up your very own snips to use automatic code generation exactly how you like it. One can also of course scour the internet and see examples of other people's snippets to get some inspiration. A great place to start is honza's snippets.

There are of course many more plugins that help with C++ development, and as you encounter problems you might look to plugins to solve them. As an example you can find my own vim dotfiles on github.

Compiler

The next thing you need is a compiler so that you can turn you C++ source files into an executable program. Also here there are many options, but it is probably a good idea to stick to the basic ones.

  • clang

    The default C++ compiler used in Apple computers these days. It is a very good compiler currently in active development and one of the better compilers for the new features introduced in the C++11 and C++14 standard. The compiler is also very beginner friendly as it has very instructive compile time error messages. It can of course also be installed on all other normal operating systems.

  • gcc

    The default C++ compiler used in most Linux distributions these days. One of its biggest advantages is that it will be available in most of the environments you will program in. You can always count in it begin installed on the various servers you might compile your code on.

  • Visual C++

    Finally we have the standard Windows C++ compiler. This will be the default compiler in use if you install one of Microsoft's IDEs such as Visual Studio. This compiler has unfortunately been lagging behind the others for the past couple of years, and will not support all of the features of the new standards, along with the new standard libraries.
