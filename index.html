Course Description

This course provides an introduction to the C++ programming language, and aims at establishing a solid foundation on which students can stand on when tackling more advanced topics in the future. The primary focus of the lecture is to prepare the students for the "Computational Physics" course taught in the winter semester. Although having prior experience in any programming language will facilitate the learning process, students from all backgrounds and disciplines are welcome to attend. On top of the lectures themselves, the course will have a large practical component where the students themselves will have to use C++ to solve various problems.

The course will cover the following topics:

  • The basics of programming
  • Tools and compilers
  • Data types, pointers and references
  • C++ memory management
  • Control structures and functions
  • Exception handling
  • Classes, inheritance and polymorphism
  • Short overview of Templates and the Standard Template Library

Learning Resources

Beginner

  • Programming -- Principles and Practice Using C++ book

    One of the better introductory books trying to teach both the basics of programming and the C++ language at the same time. Although a bit lengthy, this book teaches all the basics of C++, and it even does so without first teaching raw C. This book is also available both as a book and as an ebook at the university library.

  • C++ Pocket Reference book

    A very condensed reference for the C++ language that lists all of the C++ syntax in a convenient, easy to read form. This reference is quite handy to have early on when one still isn't that used to the syntax.

Advanced

  • Effective C++ book and More Effective C++ book

    One of those books every serious C++ programmer should read at some point in his life. The books present various methods for improving your C++ programs, most of which might be things one have never even considered.

  • Design Patterns: Elements of Reusable Object-Oriented Software book

    The starting point for anyone who wants to learn about design patterns, and any serious programmer should know about some basic design patterns to make their programs maintainable, reusable and debuggable.

  • Sutter's Mill

    Blog of C++ extraordinaire Herb Sutter, who among other achievements have been a chair in the ISO C++ Standards Committee for over a decade. His blog is enlightening for everyone who wants to keep up with the development of the C++ language. He also has a set of entries called "Guru of the Week" (GotW), where he talks about good coding practices in C++, and explains how some of the new functionalities in C++11/14 should be used.

  • C++ User Group

    We have our own C++ user group here at the university. The group meets every last Wednesday of the month for a talk given by one of the members followed by a discussion. One can find slides from previous talks on their homepage, and everyone is welcome to join for the meetings.

Lookup Resources

A couple of good places to start looking if you are having issues in C++ that you need to find a solution to

  • cppreference.com

    The most detailed C++ reference out there and a very good place to look for both the quirks of the language as well as function signatures, class interfaces and a list of everything available in the std libraries.

  • cplusplus.com

    Another C++ reference which is slightly more pedagogical than cppreference, but not as easy to use on a daily basis and not as thorough.

  • C++ Super-FAQ

    An FAQ written by some of the most prolific people in the C++ community. This FAQ is extremely extensive, and a very good lookup tool for finding out about details of the language, such as how does namespaces and variable shadowing work for template inheritance.

  • Stack Overflow

    Probably the best place to get all your C++ questions answer. One should however first see if it has been already asked as most questions have to some extent.

  • Google

    Every programmer's most valuable tool...
